# Spending Review

A local Streamlit dashboard for categorising credit card transactions from uploaded statements. Privacy-first: runs entirely on your machine, no external data transmission, configs and data gitignored.

## Features

- Upload one or more credit card statements (Excel or CSV) and produce a categorised view of spending
- Categorise transactions by mapping descriptions to categories using a curated partial-string → category table (longest match wins, case-insensitive)
- Robust header detection — finds the Date / Description / Amount columns by name, not position
- Multi-file upload with deduplication on (date, amount, description)
- Refunds and credits dropped automatically; surfaces count for transparency
- Two Excel outputs: full categorised view and unmapped subset (used to grow the mapping table)
- Pluggable parser architecture — add new statement formats without changing other code

## Setup

1. Open a terminal in this folder.

2. Create and activate a virtual environment:

   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

3. Install dependencies:

   ```bash
   pip install -r requirements.txt
   ```

4. Create the `config/` folder and populate it (see **Configuration** below).

## Running

```bash
streamlit run app.py
```

A browser tab will open at http://localhost:8501. Stop with `Ctrl+C`.

## Configuration

The `config/` folder is gitignored — a fresh clone will not run until you create it.

### `config/categories.txt`

One category per line. The authoritative list — every category used in the mapping table must appear here. `Uncategorised` is reserved and auto-assigned to anything that doesn't match.

### `config/mapping.xlsx`

Two columns:

| partial_string | category |
|---|---|
| ntuc fairprice | Groceries |
| grab | Transport |
| starbucks | Dining |

When a transaction description contains any partial_string as a substring, it gets that category. Matching is case-insensitive, longest match wins.

### `config/mapping.json`

Generated by `build_mapping.py`. Don't edit by hand — your changes will be overwritten next time you build.

## Workflow

**One-time setup:**

1. Create `config/categories.txt` with your category list
2. Create `config/mapping.xlsx` with initial partial-string → category rows
3. Run `python build_mapping.py` to generate `mapping.json`

**Monthly review:**

1. `streamlit run app.py`
2. Upload statement file(s), click **Compile**
3. Review the categorised view and category breakdown
4. Download the unmapped Excel
5. Add new partial strings to `mapping.xlsx`, save
6. Run `python build_mapping.py` to regenerate the config
7. Click **Compile** again in the dashboard to see updated categorisation

## Adding a new statement format

1. Create `parsers/format_b.py` exposing a `parse(file_bytes, filename) -> pd.DataFrame` function returning columns `date`, `description`, `amount`, `source_file`
2. Register it in the `PARSERS` dict at the top of `app.py`

The dashboard format selector picks it up automatically.
